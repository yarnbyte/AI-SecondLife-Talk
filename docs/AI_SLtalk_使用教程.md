欢迎使用 **AI SLtalk**！这是一款专为 Second Life 打造的无感全自动 AI 翻译辅助工具。它可以通过静默读取 Firestorm 的控制台日志，或通过 LSL 脚本进行中继，为你实现无缝顺滑的双向翻译体验。

## 1. 基础准备工作

### 1.1 获取 LLM API 密钥
软件的翻译引擎由大语言模型（LLM）驱动，你需要拥有一个兼容 OpenAI 格式的 API Key（例如 OpenAI 官方，或 DeepSeek、通义千问等国内各种模型的 API）。
- 获取到 **Base URL**（例如：`https://api.deepseek.com/v1`）
- 获取到 **API Key**（例如：`sk-xxxxxxx...`）
- 确认目标 **模型名称/Model**（例如：`deepseek-chat` 或 `gpt-4o-mini`）

### 1.2 准备 Firestorm 客户端设置
为了让本地翻译功能正常生效，你需要确保已在 Firestorm 中开启聊天记录保存：
1. 登录 Firestorm 客户端。
2. 打开“首选项 (Preferences)” -> “网络 & 隐私 (Network & Privacy)” ->“目录 (Directories)”。
3. 勾选 **将附近聊天记录和 IM 会话保存到你的硬盘**。

## 2. 本地日志翻译模式（推荐）

这是最简单的模式，软件直接监听你电脑上的聊天日志并输出翻译，即开即用。

### 步骤
1. **打开 AI SLtalk**，点击顶部的 ⚙️（设置）图标进入设置面板。
2. **填写 Firestorm 日志目录**：
   - 默认路径通常为 `%AppData%\Firestorm_x64`（软件会自动检测这一路径，如果为空，请手动点击“浏览”选择该文件夹）。
3. **选择 SL 账号**：
   - 路径选中后，下拉列表中会自动出现你登录过的账号文件夹（例如 `firstname_lastname`），选中你需要翻译的账号。
4. **填写 API 信息**：
   - 填入你刚才准备好的 API Key、Base URL 以及模型名称。
5. **设置翻译偏好**：
   - **翻译目标语言**：填你想看到的语言，如 `Simplified Chinese`。
   - **上文参考条数**：建议保持默认 5 条，有助于 AI 结合语境翻译的更精准。
6. **开启监听**：
   - 关闭设置面板，点击软件顶栏的“▶️ 开启监听”按钮。只要有人给你发信息，或者在此频道发言，都会自动翻译并呈现在列表中。要直接回复时，在底栏键入自己的中文即可自动翻译并**复制进剪贴板**，之后去游戏里 Ctrl+V 即可回应。

## 3. 公屏 LSL 脚本中继模式（进阶·搭配 ngrok）

有时由于本地日志有延迟等原因，或者你想做一个 HUD，在网吧/多设备玩游戏时，可以直接将公开频道里的聊天内容高速推送到中继口以加速翻译呈现。

### 原理简述
我们在 SL 里佩戴一个小部件（HUD），通过里面的 LSL 脚本实时将附近聊天使用 HTTP POST 推送到你的局域网。由于 SL 服务器在海外，你的电脑在内网，我们必须使用内网穿透（这里以 `ngrok` 为例）。

### 3.1 下载并配置 ngrok
1. 前往官网 [ngrok.com](https://ngrok.com/) 注册一个免费账号。
2. 登录平台后，下载对应系统的客户端压缩包。
3. 解压获得 `ngrok.exe`。
4. 在刚才解压 `ngrok.exe` 的文件夹里，打开 CMD/PowerShell 窗口，运行平台分配给你的 `authtoken` 进行鉴权配置：
   ```bash
   ngrok config add-authtoken 你的token码
   ```

### 3.2 启动内网穿透并映射 29853 端口
*(AI SLtalk 在底层的 LSL 侦听端口已经固定在稳定的 `29853`，你不要做任何其余修改)*
继续在命令提示符里输入：
```bash
ngrok http 29853
```
运行后，你的控制台将显示类似下面的界面：
```text
Forwarding                    https://1a2b-3c4d.ngrok-free.app -> http://localhost:29853
```
将屏幕上显示的这串 `https` 开头的链接复制下来。

### 3.3 在 AI SLtalk 内启动接收通道
1. 切换回 AI SLtalk 软件设置页，下拉勾选开启 **🛸 开启 LSL HUD 公共频道中继**。
2. 在出现的配置行中，有一个输入框提示 *可填例如 https://xxx.ngrok.io*，请将刚刚 ngrok 提供的 **公网网址粘贴在输入框内**并保存设置。
3. 软件会同时显示“重启中继服务”的按钮，可以点击一次确保通道顺畅。

### 3.4 往 Second Life 添加 LSL 接收脚本
1. 在 AI SLtalk LSL 设置区点击 **📝 复制 LSL 脚本到剪贴板**（此操作将会把附带了上述 ngrok 转发地址的一整套脚本注入剪贴板）。
2. 在 SL 游戏世界里，执行以下操作：
   - 右键“创建 (Build)”在地上建立一个基础方块（推荐将其挂载为 HUD，即在背包中 Attach to HUD 位）。
   - 编辑这个物品，在内容 (Content) 页面点 **New Script (新建脚本)**。
   - 打开脚本，全选所有默认内容，**粘贴 (Ctrl+V)** 刚好复制的代码后点击保存。
3. 佩戴该物品后，你即会收到一行 `[AISLtalk] 已开启附近频道监听...` 的提示。

**恭喜！** 现在周围任何人说话，HUD 都会在毫秒级将原文推送通过 ngrok 中转发送进你的 AI SLtalk 里进行秒翻！

## 4. 常见问题 QA

**Q：为什么开启了 LSL 中继，但我没有看见有人翻译？**
A：请确认 Ngrok 代理界面未关闭；请确认你输入到 AI SLtalk 的 ngrok 公网链接没有拼写错误（比如缺少 `https://`）。以及检查你所在的区域是否允许运行脚本。

**Q：为什么我发送信息也会有残留或者发给别人自己翻译一遍？**
A：AI SLtalk 默认过滤发送者名为你登录用户的字符和系统公告（例如包含 "Second Life"）。如果发生这种情况，可以在标签右边的 🔔 (铃铛) 图标点击静音频道加入黑名单拦截。

**Q：群组记录到底开不开？**
A：在设置里默认是关闭群组记录的。如果有经常阅读火热公租聊天频道的习惯，可以勾选，但这将消耗大量 API 额度，建议按需按量启动。
